## Phase 1: การกำหนดวัตถุประสงค์ประจำปี (KPI Target Setting)

ขั้นตอนการเริ่มต้นกำหนดเป้าหมายและดัชนีชี้วัดของแต่ละแผนกประจำปีปัจจุบัน (Current Year) หากแผนกใดสัญญายังไม่เสร็จสิ้นจะแสดงสถานะเป็น **Inactive** และเมื่อกระบวนการอนุมัติเสร็จสิ้นจะเปลี่ยนเป็นสถานะ **Active**

### 1. หน้าแดชบอร์ดและการตรวจสอบสถานะ (Dashboard & Status Tracking)
* **การกรองข้อมูล (Data Filtering):** ระบบจะแสดงผลข้อมูลและคัดกรองตามปีปัจจุบันโดยอัตโนมัติ (Default: ปี 2026)
* **รายการแผนก (Department Directory):** แสดงรายชื่อทุกแผนกในองค์กร โดยระบุสถานะชัดเจน:
  * <span style="color:#d9534f; font-weight:bold;">● Inactive</span> : แผนกที่ยังไม่ได้ระบุ/อยู่ระหว่างการตั้งวัตถุประสงค์ประจำปี
  * <span style="color:#5cb85c; font-weight:bold;">● Active</span> : แผนกที่กำหนดวัตถุประสงค์และได้รับการอนุมัติเสร็จสิ้นแล้ว

### 2. การบันทึกข้อมูลวัตถุประสงค์ของแผนก (Department Input Form)
แต่ละแผนกสามารถสร้างวัตถุประสงค์หลักได้ **หลายวัตถุประสงค์ (One-to-Many Relationship)** โดยในหนึ่งวัตถุประสงค์ต้องระบุข้อมูลให้ครบถ้วนดังนี้:

| ฟิลด์ข้อมูล (Field Name) | รูปแบบข้อมูล (Data Type / Options) | รายละเอียดและเงื่อนไขการใช้งาน |
| :--- | :--- | :--- |
| **วัตถุประสงค์ / เป้าหมาย** | Text / String | ข้อความระบุวัตถุประสงค์หลักของตัวชี้วัด |
| **เป้าหมาย (Target)** | Number / Percentage | ค่าตัวเลขเป้าหมาย เช่น จำนวน (100) หรือ เปอร์เซ็นต์ (95%) |
| **ความถี่ (Frequency)** | Dropdown Option | เลือก `ทุกเดือน` หรือ `รายไตรมาส` <br>*(หมายเหตุ: แม้เลือกรายไตรมาส แต่ระบบบังคับให้รายงานผลรายเดือน)* |
| **สูตรการคำนวณ** | Text / Formula Expression | วิธีการหรือสูตรคณิตศาสตร์ที่ใช้ประมวลผลค่า KPI |
| **แนวทางการดำเนินการ** | Text Area | รายละเอียดแผนงานหรือ Action Plan เพื่อให้บรรลุเป้าหมาย |
| **บันทึกข้อมูล/เอกสารอ้างอิง** | Text / Multi-select Tag | รหัสหรือชื่อเอกสารควบคุมที่ใช้บันทึก เช่น <br>- `FM-MR-03` (เอกสารกรอก KPI รายเดือน)<br>- `FM-IT-07` (เอกสารขึ้นทะเบียนภายในแผนก)<br>- เอกสารอื่นๆ |

### 3. ขั้นตอนการลงนามและกระบวนการอนุมัติ (Sign & Approval Workflow)
เมื่อแผนกบันทึกและตรวจสอบข้อมูลวัตถุประสงค์ทั้งหมดเรียบร้อยแล้ว จะต้องดำเนินงานตามขั้นตอนการอนุมัติดังนี้:

```
[ กดปุ่มส่ง (Submit) ] ──► [ Modal ลายเซ็นผู้จัดทำ ] ──► [ ระบุ Reviewer & Approver ] ──► [ ส่งอีเมลอนุมัติตามลำดับ ]
```

1. **การส่งข้อมูล (Submission):** ผู้ใช้กดปุ่มส่งข้อมูล ระบบจะเปิดหน้าต่างป๊อปอัพ (Modal) เพื่อให้ **ผู้จัดทำ (Creator / ตัวเรา)** ลงนามรับรอง
2. **รูปแบบการลงนาม (Signature Methods):** รองรับการลงนาม 4 รูปแบบ:
   * **Sign:** การวาดลายเซ็นสดบนหน้าจอ (Digital Pad)
   * **Typing:** การพิมพ์ชื่อ-นามสกุล โดยระบบจะแปลงเป็นฟอนต์ตัวเขียน
   * **Upload Image:** การอัปโหลดไฟล์รูปภาพลายเซ็น (PNG/JPG)
   * **Reused:** การดึงไฟล์ลายเซ็นล่าสุดที่เคยบันทึกไว้ในระบบกลับมาใช้ใหม่
3. **การกำหนดผู้ตรวจสอบและผู้อนุมัติ (Workflow Setup):** หลังจากผู้จัดทำลงนามเสร็จสิ้น ระบบจะแสดงฟิลด์ให้เพิ่มรายชื่อ:
   * **Reviewer (ผู้ทบทวน):** ตรวจสอบความถูกต้องของโครงสร้าง KPI
   * **Approver (ผู้อนุมัติ):** อนุมัติการใช้ KPI ประจำปี
4. **ระบบแจ้งเตือนผ่านอีเมล (Email Notification):** ระบบจะส่งอีเมลแจ้งเตือนพร้อมแนบลิงก์ (Secure Link) ไปยัง Reviewer และ Approver ตามลำดับ เพื่อให้เข้ามาลงนามและกดอนุมัติในระบบ

## Phase 2: การติดตามและรายงานผลรายเดือน (Monthly KPI Tracking)

หลังจากขั้นตอนใน Phase 1 ได้รับการอนุมัติ (Approved) เป็นที่เรียบร้อย สถานะของแผนกจะเปลี่ยนเป็น **Active** และระบบจะเปิดสิทธิ์ให้เข้าสู่กระบวนการรายงานผลรายเดือน

### 1. รายการตรวจสอบผลงานประจำเดือน (Monthly Update Sheet)
* ระบบจะทำการดึงข้อมูล **"ทุกวัตถุประสงค์"** ที่ระบุไว้ใน Phase 1 นำมาสร้างเป็นรายการประเมินผลแยกรายเดือน (มกราคม - ธันวาคม)
* ผู้ใช้ต้องเข้ามาอัปเดตสถานะความสำเร็จและกรอกตัวเลขผลงานจริงภายในระยะเวลาที่กำหนดของแต่ละเดือน

### 2. การบันทึกผลงานรายเดือน (Monthly Data Input)
ในทุกสิ้นเดือน เจ้าหน้าที่ผู้รับผิดชอบต้องเข้ามากรอกข้อมูลดังต่อไปนี้:
* **ผลงานจริงที่ทำได้ (Actual Performance):** กรอกค่าตัวเลขหรือเปอร์เซ็นต์ที่เกิดขึ้นจริง
* **ตรวจสอบความสอดคล้อง (Target Validation):** ระบบจะคำนวณเปรียบเทียบกับค่าเป้าหมาย (Target) ใน Phase 1 โดยอัตโนมัติ เพื่อแสดงผลว่า "บรรลุเป้าหมาย" หรือ "ไม่บรรลุเป้าหมาย"
* **เอกสารแนบหลักฐาน (Evidence Attachments):** อัปโหลดไฟล์เอกสารหลักฐานอ้างอิงเพื่อยืนยันที่มาของตัวเลขผลงานจริง (ถ้ามี) เช่น ไฟล์ Excel, PDF รายงานประจำเดือน

### 3. ขั้นตอนการลงนามและกระบวนการอนุมัติรายเดือน (Monthly Sign-off Workflow)
```
[ บันทึกและแนบไฟล์ ] ──► [ Modal ลายเซ็นผู้จัดทำ ] ──► [ ระบุ Approver รายเดือน ] ──► [ ส่งอีเมลให้ Approver เซ็นชื่อ ]
```

1. **การส่งรายงานผล (Monthly Submit):** เมื่อกรอกข้อมูลและตรวจสอบไฟล์แนบครบถ้วนแล้ว กดปุ่มส่งรายงาน
2. **การลงนามผู้รายงาน (Creator Signature):** ระบบจะแสดงหน้าต่างป๊อปอัพ (Modal) เพื่อให้ผู้จัดทำลงนาม โดยสามารถเลือกรูปแบบลายเซ็นได้ 4 วิธีเช่นเดียวกับ Phase 1 (`Sign`, `Typing`, `Upload Image`, `Reused`)
3. **การระบุผู้อนุมัติ (Approver Selection):** เมื่อผู้จัดทำลงนามเสร็จสิ้น ระบบจะแสดงช่องให้กรอกหรือระบุรายชื่อ **Approver (ผู้อนุมัติผลประจำเดือน)**
4. **การส่งอีเมลอนุมัติ (Email Sign-off):** ระบบจะดำเนินการส่งอีเมลแจ้งเตือนอัตโนมัติไปยัง Approver เพื่อให้คลิกลิงก์เข้ามาตรวจสอบรายงานผลประจำเดือน เอกสารแนบ และลงนามอนุมัติ (Sign) ปิดรอบของเดือนนั้นๆ
