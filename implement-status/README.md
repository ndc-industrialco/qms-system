# Implement Status

Use this folder to record implementation progress during development.

For the audit module, use:

- `implement-status/audit-module.md`

Every meaningful implementation batch must update the status file.

Minimum required content per entry:

- date/time
- implementer
- task or batch name
- summary of work
- implementation details
- files touched
- tests run
- blockers or follow-up items

Do not rely on git history alone.
